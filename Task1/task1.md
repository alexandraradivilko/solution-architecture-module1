# Архитектурное видение

## Задача

Проблема: 
Компания «Diamond Universe» столкнулась с необходимостью перевести процесс продажи алмазов в цифровой формат. 
Текущая модель продаж основана на офлайн-аукционах, где покупатели лично оценивают камни перед покупкой. 
Из-за ограничений, вызванных пандемией, такой формат перестал обеспечивать устойчивость бизнеса и масштабируемость продаж.

Задача: 
Необходимо организовать цифровой канал реализации алмазов для сохранения объёмов продаж, который позволит:
- публиковать данные о камнях для удалённого анализа покупателями;
- проводить онлайн-аукционы;
- обеспечивать безопасную логистику и сопровождение сделок;
- сохранить уровень доверия к качеству оценки алмазов.

Решение должно помочь:
- бизнесу — сохранить и расширить рынок сбыта;
- покупателям — участвовать в торгах удалённо;
- внутренним подразделениям — организовать прозрачный и управляемый процесс подготовки цифровых аукционов.

---

## Контекст

### Причины возникновения задачи

В 2020 году рынок драгоценных камней столкнулся с резким сокращением офлайн-продаж и нарушением международной логистики. 
Для компании «Diamond Universe» это привело к невозможности использовать традиционный формат очных аукционов как основной канал реализации алмазов.
При этом покупатели всё ещё нуждаются в возможности оценки качества камней перед покупкой. Для этого требуется организовать цифровую оцифровку алмазов и предоставить клиентам удалённый доступ к данным.
Дополнительным фактором является жёсткое ограничение по срокам: первый цифровой аукцион необходимо провести через два месяца.

### Текущее состояние (As-Is)

В компании используются следующие информационные системы:
- ИС «Алмазы» — учёт алмазов и подготовка ведомостей;
- ИС «Аукцион» — проведение офлайн-аукционов;
- ИС «Логистика» — сопровождение перевозок;
- ИС «Бухгалтерия» — контрактование и оплата услуг.

Процесс цифровой продажи отсутствует:
- нет цифрового представления алмазов;
- отсутствует интеграция с лабораторией;
- нет механизма публикации цифровых данных для клиентов;
- онлайн-аукцион не поддерживается;
- обмен данными между системами для нового процесса не автоматизирован.

### Целевое состояние (To-Be)

Компания должна получить возможность:
- отбирать алмазы для цифровых аукционов;
- передавать их в лабораторию для оцифровки;
- публиковать результаты оцифровки клиентам;
- проводить онлайн-аукционы;
- автоматически сопровождать логистику и расчёты с подрядчиками.

Целевое решение предполагает:
- автоматизацию процессов подготовки и проведения аукциона;
- интеграцию внутренних систем с ИС «Лаборатория»;
- отказ от Excel и ручного переноса данных;
- использование защищённых каналов передачи информации;
- снижение зависимости от внешних облачных сервисов и отдельных сотрудников.

### Ограничения

Ключевые ограничения проекта:
- срок запуска первого цифрового аукциона — 2 месяца;
- ошибки ручного ввода/переноса данных через Excel;
- необходимость работы с внешней лабораторией;
- зависимость части процессов от внешних подрядчиков.

## Описание оптимального решения

### Подход к реализации

Для выполнения бизнес-задачи предлагается реализовать решение в два этапа:
1. Промежуточное решение для быстрого запуска цифрового аукциона.
2. Целевое решение с полной автоматизацией процессов.

Такой подход позволит соблюсти сроки запуска и одновременно сформировать основу для дальнейшего развития архитектуры.

### Этап 1. Промежуточное решение

Цель этапа — провести первый цифровой аукцион через 2 месяца с минимальными изменениями существующего ИТ-ландшафта.

Предлагается:
- использовать существующие ИС без значительных доработок;
- организовать обмен данными через утверждённые Excel-шаблоны;
- подключить внешнюю ИС «Лаборатория»;
- использовать частное облако для публикации данных клиентам;
- выполнять часть операций вручную под контролем сотрудников.

Основные процессы:
1. Отбор алмазов в ИС «Алмазы».
2. Передача камней через ИС «Логистика» в лабораторию.
3. Оцифровка алмазов внешней лабораторией.
4. Передача результатов в ИС «Аукцион».
5. Проведение онлайн-аукциона.
6. Организация доставки победителям торгов.

Недостатки подхода:
- высокая доля ручных операций;
- риск ошибок при переносе данных;
- зависимость от внешнего облака и сотрудников.

### Этап 2. Целевое решение

После запуска первого аукциона предлагается перейти к поэтапной автоматизации.

Основные изменения:
- автоматизация интеграций между ИС;
- отказ от Excel;
- внедрение API-взаимодействия;
- организация защищённых каналов передачи данных;
- перенос облачных сервисов на инфраструктуру компании;
- централизованный мониторинг статусов подготовки аукционов.

Целевая интеграционная схема:
- ИС «Алмазы» → ИС «Логистика»;
- ИС «Лаборатория» → ИС «Аукцион»;
- ИС «Лаборатория» → ИС «Бухгалтерия»;
- ИС «Логистика» → ИС «Бухгалтерия».

### Необходимые ресурсы

Для реализации решения потребуются:
- команда разработки и интеграции;
- специалисты по информационной безопасности;
- представители бизнес-подразделений;
- подрядчик лаборатории;
- поставщик облачной инфраструктуры;
- специалисты по сопровождению логистики.

### Основные риски

Основные риски проекта:
- задержка интеграции с лабораторией;
- утечка конфиденциальных данных;
- ошибки ручного ввода;
- нестабильность внешних сервисов;
- зависимость от облачного провайдера;
- срыв сроков автоматизации.

Для снижения рисков предлагается:
- разделить проект на этапы;
- минимизировать объём изменений в первом релизе;
- использовать временные ручные процессы;
- предусмотреть аудит действий пользователей;
- организовать контроль качества данных.

### Оценка сроков

Предварительная оценка:
- промежуточное решение — 2 месяца;
- автоматизация и развитие целевого решения — 6–12 месяцев поэтапно.

Приоритетом проекта является успешный запуск первого цифрового аукциона в установленный срок с последующим переходом к устойчивой целевой архитектуре.
